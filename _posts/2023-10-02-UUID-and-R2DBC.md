Uuid is not a primitive type in MySQL and R2DBC does not support it :(.

A workaround is
writing [R2dbcCustomConversions](https://stackoverflow.com/questions/64351344/r2dbc-custom-converter),
but it still cannot fetch the id generated by MySQL uuid().

Finally, the id is initialized in a BeforeConvertCallback (similar to @PrePersist).

Tests on PostgreSQL shows that it works properly with uuid and r2dbc without any extra setup. So,
please use PostgreSQL if you try webflux & r2dbc.
